Project 1: Capsule Neural Networks for Animal/Human Pose Estimation

Advisor: Georgios Leontidis
Project Areas: AI / ML

Description: Capsule Neural Networks (CapsNet) have been shown to be better learners of object properties in images compared to convolutional neural networks. There are several theories as to why that is the case but the routing mechanism that models the part-whole relationships helps to capture [viewpoint equivariance] (https://arxiv.org/pdf/2206.02664.pdf). This project will use open datasets to develop a CapsNet model to estimate animal/human pose. The student will explore the suitability of several open datasets that exist in this area and upon deciding which one to use they will then go on to develop a CapsNet model based on a few routing mechanisms, e.g. [VarCaps] (https://ojs.aaai.org/index.php/AAAI/article/download/5785/5641). A baseline ResNet 50 CNN model will be used to compare with all variations of CapsNet models. It is anticipated that the student will work alongside a current PhD student, and target a journal/conference paper publication whilst writing up the L4 dissertation.

Kewords: Capsule Neural Networks; Pose Estimation; Computer Vision; Machine Learning

Preliminary Reading

[Introducing Routing Uncertainty in Capsule Neural Networks] (https://proceedings.neurips.cc/paper/2020/file/47fd3c87f42f55d4b233417d49c34783-Paper.pdf)

Project 9: Hierarchical Goal Network Planning

Advisor: Felipe Meneguzzi
Project Areas: AI / General

Description: While classical planning in state spaces require minimal domain engineering, but can be inefficient for domains in where domain-independent heuristics provide uninformed cost estimates. By contrast Hierarchical Task Network (HTN) planning allow domain engineers to encode a substantial amount of guidance for the underlying algorithms. This comes at the cost of of substantial domain engineering, and the risk of an engineer to provide domain knowledge that turns the problem undecidable. Hierarchical Goal Network (HGN) mitigate this problem by merging both approaches using a new formalism. This formalism, however, has no currently open sourced planners available. Thus this project consists of implementing an HGN planner using a formalism like GoDel, ideally with Heuristics for HGN Planning, and cost-optimality guarantees

Kewords: Automated Planning, Hierarchical Planning

Preliminary Reading

  • The GoDeL Planning System: A More Perfect Union of Domain-Independent and Hierarchical Planning
  • Incorporating Domain-Independent Planning Heuristics in Hierarchical Planning
  • Cost-Optimal Algorithms for Planning with Procedural Control Knowledge

Project 36: Multi-agent planning in coordination-intensive domains

Advisor: Rafael C. Cardoso
Project Areas: AI / General

Description: Single-agent (centralised) planning has been shown to suffer from scalability issues when dealing with problems containing a large number of agents and goals. Multi-agent planners are often domain specific to deal with the constraints and assumptions from different application domains. A general approach to multi-agent planning usually tackles this problem by performing factorisation techniques to split it into sub-problems and then using a single-agent planner to solve them. A recent multi-agent planner called EgoPlan has been proposed for temporal problems (where actions have durations). In this project, you will engineer coordination-intensive domains (based on existing domains) to further evaluate EgoPlan and attempt to categorise how coordination/interaction can be identified in planning domains and problems, as well as looking into creating a metric for evaluating how much coordination/interaction the planner made use of. Optional objectives include expanding this study to other multi-agent planners.

Requirements:
- Previous knowledge of Python for interacting with the various Python APIs - Previous knowledge of automated planning (PDDL and temporal planning) is desirable, but not mandatory

Kewords: automated task planning, PDDL, multi-agent planning

Preliminary Reading
